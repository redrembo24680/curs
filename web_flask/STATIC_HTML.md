# Статичні HTML файли

Проект тепер використовує статичні HTML файли замість Jinja2 templates.

## Структура

```
web_flask/static/
├── index.html          # Головна сторінка
├── matches.html        # Матчі та голосування
├── players.html        # Список гравців
├── stats.html          # Статистика
├── health.html         # Стан API
├── login.html          # Авторизація
├── register.html       # Реєстрація
├── app.js              # API клієнт та утиліти
├── matches.js          # Логіка для сторінки матчів
├── style.css           # Основні стилі
├── modal.css           # Стилі для модальних вікон
└── stats.css           # Стилі для статистики
```

## Як це працює

1. **Flask routes** віддають статичні HTML файли через `send_from_directory()`
2. **JavaScript** завантажує дані через API (`app.js` містить API клієнт)
3. **Всі дані** отримуються динамічно через C++ backend API
4. **Авторизація** поки що не реалізована для статичних HTML (потрібен Flask для сесій)

## Переваги

- ✅ Швидше завантаження (статичні файли кешуються браузером)
- ✅ Можна відкривати HTML файли напряму (без Flask)
- ✅ Простіше деплой (можна використовувати будь-який веб-сервер)
- ✅ Краще для CDN та статичного хостингу

## Обмеження

- ⚠️ Авторизація поки що не працює (потрібен Flask для сесій)
- ⚠️ Голосування потребує авторизації через Flask
- ⚠️ Адмін функції (додавання команд/гравців) потребують авторизації

## Використання

Всі сторінки доступні через Flask:
- `http://localhost:5000/` - головна
- `http://localhost:5000/matches` - матчі
- `http://localhost:5000/players` - гравці
- `http://localhost:5000/stats` - статистика
- `http://localhost:5000/health` - стан API

Або можна відкривати HTML файли напряму з папки `static/` (якщо CORS дозволено).
